You will use the MNIST audio dataset. It is an open source dataset and you can download it from kaggle.

  • The dataset consists of 30000 audio samples of spoken digits (0-9) of 60 folders and 500 files each.
  • There is one directory per speaker holding the audio recordings.

Use the following line of code to load the audio files

You will use the MFCC features for representing the audio.
MFCCs are a common feature representation for audio classification in machine learning. They capture the spectral envelope of the sound by converting the audio signal to the frequency domain, applying a Mel-scale filterbank, taking the logarithm of the filterbank energies and applying DCT. This results in a compact, low-dimensional representation that captures important spectral information and is used as input features for training models for tasks such as speech recognition, music genre classification and sound event detection.
Dont worry if none of this makes any sense, you can still use them as your features. But if you want to understand them in more detail you can read about them from here.
Length of each feature vector will be $n$, where $n$ is the number of mfcc features (out of a total of 40) you decide to use.
Your dataset will be a $m \times (n+1$) matrix, where each row will represent an audio file and each column wil represent 1 feature. The last column will be the label of the digit spoken.

# Use this code to extract MFCC features from audio file

def get_MFCC(audio, sr, numFeatures):
    features = psf.mfcc(audio, sr, 0.025, 0.01, numFeatures, appendEnergy = True)
    return np.mean(features, axis=0)

Question: The features we get here are numerical, what if we also had categorical features e.g., even vs odd digit. How would knn deal with those?¶

Ans: Since the classifier works on the basis of finding minimal distance between two points, it does not matter if the data is numerical or categorical. In case of ordinal data if a distance function, that relates data points by defining "closer" and "farther" unit(s), can be generated then using KNN for ordinal data is same as using it for numerical data.

def euclidean_distance(row1, row2):
    return np.linalg.norm(row1 - row2)

def manhattan_distance(row1, row2):
    return sum(abs(val1-val2) for val1, val2 in zip(row1,row2))

def my_mode(sample):
    c = Counter(sample)
    return [k for k, v in c.items() if v == c.most_common(1)[0][1]]

def custom_knn(testDataPoint, distanceMetric, k):

  distanceMatrix = []
  NNLabel = [] 
  flag = True
  positives=0
  negatives=0

  for row in trainingD: #CALCULATING DISTANCES 
    if distanceMetric == "Euclidean": 
      distanceMatrix.append(euclidean_distance(testDataPoint[0:numFeatures], row[0:numFeatures]))
    elif distanceMetric == "Manhattan":
      distanceMatrix.append(manhattan_distance(testDataPoint[0:numFeatures], row[0:numFeatures]))
    else:
      print("Please enter 'Euclidean' or 'Manhattan'")

  while flag == True and k>=1: #INCASE OF TIES, break them by backing off to k-1 value
    for x in range(k): #CALCULATING K NEAREST NEIGHBORS
      NN = (np.argsort(distanceMatrix))[x]#GETTING K-TH ROW FROM MATRIX
      NNLabel.append(int((trainingD[NN])[numFeatures])) #GETTING CORRESPOBDING LABELS FROM TRAINING DATA
      if int((trainingD[NN])[numFeatures])== int(testDataPoint[numFeatures]):
        positives=positives+1
      if int((trainingD[NN])[numFeatures]) != int(testDataPoint[numFeatures]):
        negatives=negatives+1

      #print((np.argsort(distanceMatrix))[x])
      #print((trainingD[NN])[numFeatures])
    if len(my_mode(NNLabel)) > 1 :
      flag = True
      k=k-1
    else:
      flag = False
  #print(NNLabel)
  return my_mode(NNLabel)[0],positives,negatives

Question: What can we interpret from the F1 score that we can not from accuracy?¶

Answer: ROC curve requires true positive (TP), true negative (TN), false positive (FP) and false negative (FN) points from the data. To get accuracy, we require TP and TN to figure out how "correct" the results are. For F1 score, we need FP and FN. f1 score is more crucial than accuracy since an FP, even comparatively smaller, can cause a major impact. For instance, if a patient is correctly predicted for not having cancer (TN) it means that the system is working accurately. But if the patient is wrongly predicted for not having cancer (FN) that would be a high risk.

Question: Why is standardization (feature scaling) required for KNN to work properly.¶

Ans: Since KNN works on the basis of point to point distance, it is important that the data is standardized/normalized in order to have a consistent system to work with.

Question: How does the choice of 'k' affect overfitting and underfitting?¶

Question: Briefly explain the curse of dimensionality and how PCA can be used to tackle it.¶

Ans: Curse of dimensionality refers to increase in the set of features. If the features of data increse, it would in turn make the data sparse which would cause an overfit. Since PCA reduces dimension by projecting data onto orthognal axes, it can be used to tackle this curse.
